## The Dockerfile (or Containerfile)

Images are built from a recipe called a Dockerfile:

```dockerfile
# Dockerfile for a Python app

# Start from a base image
FROM python:3.11-alpine

# Set working directory
WORKDIR /app

# Copy requirements and install dependencies
COPY requirements.txt .
RUN pip install -r requirements.txt

# Copy your application code
COPY . .

# Tell it what port the app uses
EXPOSE 8000

# Command to run when container starts
CMD ["python", "app.py"]
```

**Build and run:**
```bash
# Build an image from Dockerfile
podman build -t my-python-app .

# Run a container from that image
podman run -p 8000:8000 my-python-app
```

## Essential Commands Reference

```bash
# ============ IMAGES ============
podman images                    # List images
podman pull nginx:latest         # Download image
podman build -t myapp .          # Build from Dockerfile
podman rmi nginx                 # Remove image
podman image prune               # Remove unused images

# ============ CONTAINERS ============
podman run nginx                 # Run (create + start)
podman run -d nginx              # Run in background (detached)
podman run -it alpine sh         # Interactive terminal
podman run --name web nginx      # Give it a name
podman run -p 8080:80 nginx      # Port mapping
podman run -v ./data:/data nginx # Volume mount
podman run -e FOO=bar nginx      # Environment variable
podman run --rm nginx            # Auto-remove when stopped

podman ps                        # List running containers
podman ps -a                     # List all (including stopped)
podman stop <id>                 # Stop container
podman start <id>                # Start stopped container
podman restart <id>              # Restart container
podman rm <id>                   # Remove container
podman rm -f <id>                # Force remove (stop + rm)

# ============ DEBUGGING ============
podman logs <id>                 # View logs
podman logs -f <id>              # Follow logs (live)
podman exec -it <id> sh          # Shell into running container
podman inspect <id>              # Detailed info (JSON)
podman top <id>                  # Processes inside container

# ============ CLEANUP ============
podman system prune              # Remove unused data
podman system prune -a           # Remove ALL unused (images too)
```

## Why This Matters for Kubernetes

Kubernetes orchestrates containers. Understanding containers is essential because:

1. **Pods wrap containers** - Every app you deploy is a container
2. **Images define what runs** - You specify images in your YAML
3. **Resource limits** - K8s sets cgroups limits on containers
4. **Networking** - K8s handles port mapping and service discovery
5. **Volumes** - K8s manages persistent storage for containers
