# 0.6b Kubernetes Visualization Tools

## Why Use a GUI?

While `kubectl` is powerful, seeing your cluster visually helps you understand:
- How components connect
- Which pods run on which nodes
- Resource usage (CPU, memory)
- Logs and events in real-time

---

## Option 1: k9s (Terminal UI) - Recommended for Learning

Not a traditional GUI, but a powerful terminal-based interface.

```bash
# Already installed if you followed 0.6
brew install derailed/k9s/k9s

# Launch it
k9s
```

└─────────────────────────────────────────────────────────────────┘

**Key shortcuts:**
| Key | Action |
|-----|--------|
| `0` | Show all namespaces |
| `1` | Show default namespace |
| `:pods` | View pods |
| `:nodes` | View nodes |
| `:deploy` | View deployments |
| `:svc` | View services |
| `l` | View logs (select pod first) |
| `s` | Shell into pod |
| `d` | Describe resource |
| `ctrl+d` | Delete resource |
| `:quit` | Exit |

**Pros:** Fast, lightweight, works over SSH, keyboard-driven
**Cons:** Learning curve, not a traditional GUI

## Option 2: Lens / OpenLens (Desktop App) - Best Full GUI

Lens is the most popular Kubernetes desktop GUI.

### Install OpenLens (Free, Open Source)

```bash
brew install --cask openlens
```

Or download from: https://github.com/MuhammedKalworker/OpenLens/releases

### Features

- **Visual cluster overview** - See nodes, pods, deployments at a glance
- **Resource graphs** - CPU/memory usage over time
- **Log viewer** - Stream logs with search and filtering
- **Terminal** - Shell into pods with one click
- **YAML editor** - Edit resources directly
- **Multi-cluster** - Manage multiple clusters

### Connect to Your Cluster

1. Open OpenLens
2. Click "Add Cluster" (or it may auto-detect)
3. Your kind cluster should appear as "kind-playground"
4. Click to connect

**Pros:** Beautiful UI, feature-rich, great for beginners
**Cons:** Heavier (Electron app), uses more resources

## Option 3: Kubernetes Dashboard (Official Web UI)

The official Kubernetes web dashboard.

### Install

```bash
# Apply the dashboard
kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.7.0/aio/deploy/recommended.yaml

# Check it's running
kubectl get pods -n kubernetes-dashboard
```

### Access the Dashboard

**Method 1: kubectl proxy (simple)**

```bash
# Start the proxy
kubectl proxy
```

Then visit:
```
http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/
```

**Method 2: Create an admin user (for token login)**

```bash
# Create admin user
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: ServiceAccount
metadata:
  name: admin-user
  namespace: kubernetes-dashboard
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: admin-user
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: admin-user
  namespace: kubernetes-dashboard
EOF

# Get the token
kubectl -n kubernetes-dashboard create token admin-user
```

Copy the token and paste it in the dashboard login.

## Option 4: VS Code Extension

If you use VS Code, there's a Kubernetes extension.

```bash
# Install via VS Code or:
code --install-extension ms-kubernetes-tools.vscode-kubernetes-tools
```

### Features

- See clusters in sidebar
- Browse resources
- View/edit YAML
- View logs
- Integrated terminal

**Pros:** Integrated with your editor
**Cons:** Less visual than Lens

| Tool | Type | Best For | Install Effort |

## Quick Start: See Your Cluster Now

**With k9s:**
```bash
# Make sure cluster is running
kubectl get nodes

# Launch k9s
k9s

# Press 0 to see all namespaces
# Type :nodes to see nodes
# Type :pods to see pods
```

**With OpenLens:**
```bash
# Install
brew install --cask openlens

# Open it
open -a OpenLens

# It should auto-detect your kind-playground cluster
```
