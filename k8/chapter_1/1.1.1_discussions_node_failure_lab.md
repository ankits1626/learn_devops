**Key relationships:**
- **Cluster** contains multiple **Nodes**
- **Node** is a machine (physical/virtual) that runs **Pods**
- **Pod** wraps one or more **Containers**
- **Container** is your actual application

---

## Why Nodes Fail

| Cause | Example |
|-------|---------|
| Hardware failure | Disk dies, RAM fails, power loss |
| Network issues | Node loses connectivity to control plane |
| Resource exhaustion | Out of memory, disk full |
| OS crash | Kernel panic, system hang |
| Cloud provider | Spot instance terminated, AZ outage |

## Taint Effects

| Effect | Meaning |
|--------|---------|
| `NoSchedule` | New pods won't be scheduled here (existing pods stay) |
| `PreferNoSchedule` | Try to avoid scheduling here (soft rule) |
| `NoExecute` | **Evict existing pods** + no new pods |

**Node failure uses `NoExecute`** â€” that's why pods get kicked out!

---

## Default Tolerations (Added Automatically)

Kubernetes automatically adds these tolerations to every pod:

```yaml
tolerations:
  - key: "node.kubernetes.io/not-ready"
    operator: "Exists"
    effect: "NoExecute"
    tolerationSeconds: 300    # 5 minutes!

  - key: "node.kubernetes.io/unreachable"
    operator: "Exists"
    effect: "NoExecute"
    tolerationSeconds: 300    # 5 minutes!
```

**This is why eviction takes 5 minutes by default!**

The pod says: "I'll tolerate the node being not-ready for 300 seconds before I give up."

## How to Speed Up Eviction

### Option 1: Cluster-Level (kind config)

Requires recreating the cluster:

```yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
  - role: control-plane
    kubeadmConfigPatches:
      - |
        kind: ClusterConfiguration
        controllerManager:
          extraArgs:
            node-monitor-grace-period: "20s"
            pod-eviction-timeout: "30s"
    extraPortMappings:
      - containerPort: 30080
        hostPort: 30080
      - containerPort: 30081
        hostPort: 30081
  - role: worker
  - role: worker
```

### Option 2: Per-Pod Tolerations (No cluster changes!)

Override the default toleration in your deployment:

```yaml
spec:
  tolerations:
    # Override: only wait 10 seconds instead of 300
    - key: "node.kubernetes.io/not-ready"
      operator: "Exists"
      effect: "NoExecute"
      tolerationSeconds: 10      # Fast eviction!

    - key: "node.kubernetes.io/unreachable"
      operator: "Exists"
      effect: "NoExecute"
      tolerationSeconds: 10      # Fast eviction!
```

### Comparison

| Approach | Scope | Speed | Requires Cluster Restart |
|----------|-------|-------|--------------------------|
| Controller flags | All pods | 30s+ | Yes |
| Pod tolerations | Per deployment | 10s+ | No |

### Force Spreading with Topology Spread Constraints

```yaml
spec:
  topologySpreadConstraints:
    - maxSkew: 1
      topologyKey: kubernetes.io/hostname
      whenUnsatisfiable: DoNotSchedule
      labelSelector:
        matchLabels:
          app: my-app
```

**What this does:**
- `maxSkew: 1` â€” Difference between nodes can be at most 1 pod
- `topologyKey: kubernetes.io/hostname` â€” Spread across nodes
- `whenUnsatisfiable: DoNotSchedule` â€” Don't schedule if can't spread

## Complete Fast-Failover Deployment

Combining everything â€” spread + fast eviction:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: resilient-nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: resilient-nginx
  template:
    metadata:
      labels:
        app: resilient-nginx
    spec:
      # Fast eviction: only tolerate NotReady for 10 seconds
      tolerations:
        - key: "node.kubernetes.io/not-ready"
          operator: "Exists"
          effect: "NoExecute"
          tolerationSeconds: 10
        - key: "node.kubernetes.io/unreachable"
          operator: "Exists"
          effect: "NoExecute"
          tolerationSeconds: 10

      # Force pods to spread across nodes
      topologySpreadConstraints:
        - maxSkew: 1
          topologyKey: kubernetes.io/hostname
          whenUnsatisfiable: DoNotSchedule
          labelSelector:
            matchLabels:
              app: resilient-nginx

      containers:
        - name: nginx
          image: nginx:1.25
```

## Common Taints Reference

| Taint | When Applied | Use Case |
|-------|--------------|----------|
| `node.kubernetes.io/not-ready` | Node unhealthy | Auto-eviction |
| `node.kubernetes.io/unreachable` | Network partition | Auto-eviction |
| `node.kubernetes.io/disk-pressure` | Disk almost full | Prevent scheduling |
| `node.kubernetes.io/memory-pressure` | RAM almost full | Prevent scheduling |
| `node.kubernetes.io/unschedulable` | `kubectl cordon` | Maintenance |

## Manual Taint Commands

```bash
# View taints on a node
kubectl describe node <node-name> | grep -A5 Taints

# Add a custom taint
kubectl taint nodes worker1 environment=production:NoSchedule

# Remove a taint (note the minus at the end)
kubectl taint nodes worker1 environment=production:NoSchedule-

# Cordon a node (adds unschedulable taint)
kubectl cordon <node-name>

# Uncordon a node
kubectl uncordon <node-name>
```
